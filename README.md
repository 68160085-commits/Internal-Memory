บทที่ 5.2: การแก้ไขข้อผิดพลาด (Error Correction)กลไกการแก้ไขข้อผิดพลาดในหน่วยความจำสารกึ่งตัวนำ มีจุดประสงค์เพื่อเพิ่มความน่าเชื่อถือและความถูกต้องของข้อมูลในระบบคอมพิวเตอร์ โดยมีรายละเอียดหลัก
1. ประเภทของข้อผิดพลาดในหน่วยความจำ (Types of Memory Failures)ข้อผิดพลาดที่เกิดขึ้นกับหน่วยความจำสารกึ่งตัวนำสามารถจำแนกตามลักษณะการเกิดได้เป็น 2.ประเภทหลัก:ความล้มเหลวแบบถาวร (Hard Failure): เป็นความเสียหายที่เกิดขึ้นกับตัวอุปกรณ์ฮาร์ดแวร์โดยตรง ส่งผลให้บิตข้อมูล ณ ตำแหน่งนั้นไม่สามารถเปลี่ยนแปลงค่าได้ (ค้างอยู่ที่ 0 หรือ 1 ตลอดเวลา)ข้อผิดพลาดแบบชั่วคราว (Soft Error): เป็นเหตุการณ์สุ่มที่ทำให้ค่าของบิตข้อมูลเปลี่ยนแปลงไป (เช่น จาก 0 เป็น 1) โดยที่ตัวอุปกรณ์ไม่ได้เกิดความเสียหายถาวร มักมีสาเหตุจากปัญหาการรบกวนทางไฟฟ้าหรืออนุภาคจากรังสี
2. หลักการทำงานของรหัสแก้ไขข้อผิดพลาด (Error-Correcting Code - ECC)ระบบจะใช้วิธีการเพิ่มบิตตรวจสอบ (Check Bits) เข้าไปพร้อมกับข้อมูลหลัก เพื่อให้สามารถตรวจสอบและซ่อมแซมข้อมูลได้โดยอัตโนมัติ มีขั้นตอนดังนี้:กระบวนการจัดเก็บ (Write Operation): ระบบจะนำข้อมูลขนาด $M$ บิต มาผ่านฟังก์ชัน $f$ เพื่อสร้างรหัสตรวจสอบ (Check Bits) ขนาด $K$ บิต และจัดเก็บข้อมูลรวมกันในรูปแบบ $M + K$ บิตกระบวนการเรียกใช้ (Read Operation): เมื่อมีการอ่านข้อมูล ข้อมูล $M$ บิตจะถูกนำมาคำนวณเพื่อสร้างรหัสตรวจสอบชุดใหม่ แล้วนำไปเปรียบเทียบกับรหัสตรวจสอบเดิมที่จัดเก็บไว้ก่อนหน้า
3. ผลลัพธ์จากการเปรียบเทียบรหัสตรวจสอบ (Comparison Results)การเปรียบเทียบรหัสจะนำไปสู่ผลลัพธ์3 กรณี:กรณีปกติ (No Error Detected): รหัสทั้งสองชุดตรงกัน ระบบจะส่งข้อมูลออกไปใช้งานทันทีกรณีแก้ไขได้ (Single-bit Error Corrected): หากพบข้อผิดพลาดเพียง 1 บิต ระบบจะระบุตำแหน่งที่ผิดพลาดและทำการแก้ไขให้ถูกต้องก่อนส่งออกกรณีตรวจพบแต่แก้ไขไม่ได้ (Uncorrectable Error): หากพบข้อผิดพลาดหลายบิตเกินกว่าขีดความสามารถของรหัสที่จะแก้ไขได้ ระบบจะทำการรายงานความผิดพลาดไปยังส่วนควบคุม
4. รหัสแฮมมิง (Hamming Code) และมาตรฐาน SEC-DEDรหัสที่ได้รับความนิยมสูงสุดในระบบหน่วยความจำคือ Hamming Code ซึ่งมีคุณสมบัติรองรับมาตรฐาน SEC-DED (Single-Error-Correcting, Double-Error-Detecting):SEC: สามารถแก้ไขข้อผิดพลาดได้โดยอัตโนมัติหากเกิดขึ้นเพียง 1 บิตDED: สามารถตรวจจับได้ว่ามีข้อผิดพลาดเกิดขึ้นหากมีการผิดพลาดพร้อมกัน 2 บิต (แต่ไม่สามารถแก้ไขได้)
