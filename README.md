บทสรุปของเนื้อหา บทที่ 5.2 เรื่อง Error Correction จากหนังสือ Computer Organization and Architecture ของ William Stallings  มีประเด็นสำคัญดังนี้ครับ:1. ความจำเป็นของ Error Correctionหน่วยความจำแบบสารกึ่งตัวนำ (Semiconductor Memory) อาจเกิดข้อผิดพลาดได้จากปัจจัยภายนอก:Hard Failure: ความเสียหายถาวรที่ตัวฮาร์ดแวร์ ทำให้บิตข้อมูลค้างอยู่ที่ค่าเดิมตลอด (0 หรือ 1) Soft Error: ข้อผิดพลาดชั่วคราวที่ทำให้ข้อมูลเปลี่ยนไป (เช่น จาก 0 เป็น 1) โดยที่ฮาร์ดแวร์ไม่ได้เสียหาย มักเกิดจากปัญหาพลังงานหรืออนุภาคจากรังสี 2. กลไกการทำงานของรหัสแก้ไขข้อผิดพลาด (Error-Correcting Code - ECC)ระบบจะใช้การเพิ่ม "บิตตรวจสอบ" (Check Bits) เข้าไปพร้อมกับข้อมูลหลัก เพื่อให้สามารถตรวจจับและแก้ไขข้อผิดพลาดได้อัตโนมัติ:กระบวนการเขียน (Write): เมื่อจะเก็บข้อมูลขนาด $M$ บิต ระบบจะใช้ฟังก์ชัน $f$ สร้างรหัสตรวจสอบขนาด $K$ บิต และจัดเก็บรวมกันเป็น $M + K$ บิต กระบวนการอ่าน (Read): เมื่ออ่านข้อมูลออกมา ระบบจะนำข้อมูล $M$ บิตที่อ่านได้มาสร้างรหัสตรวจสอบชุดใหม่ แล้วนำไปเปรียบเทียบกับรหัสเดิมที่เก็บไว้ 3. ผลลัพธ์จากการตรวจสอบการเปรียบเทียบรหัสตรวจสอบจะให้ผลลัพธ์ 3 รูปแบบ:ไม่มีข้อผิดพลาด: ข้อมูลถูกส่งออกไปใช้งานทันที ตรวจพบและแก้ไขได้: หากเป็นข้อผิดพลาดเพียงบิตเดียว (Single-bit error) ระบบจะรู้ตำแหน่งและสลับค่าบิตนั้นให้ถูกต้องก่อนส่งออก ตรวจพบแต่แก้ไขไม่ได้: หากมีข้อผิดพลาดหลายบิตเกินกว่าที่รหัสจะรับรอง ระบบจะรายงานว่าเป็นข้อผิดพลาด 4. รหัสที่นิยมใช้ (Hamming Code)รหัสที่ถูกพูดถึงมากที่สุดคือ Hamming Code ซึ่งเป็นรหัสพื้นฐานที่ช่วยให้ระบบสามารถทำ SEC-DED (Single-Error-Correcting, Double-Error-Detecting) คือสามารถแก้ไขข้อผิดพลาด 1 บิต และตรวจจับข้อผิดพลาด 2 บิตได้ในเวลาเดียวกัน สรุปสั้นๆ: บทที่ 5.2 เน้นไปที่วิธีการทำให้หน่วยความจำมีความน่าเชื่อถือสูงขึ้น โดยการใช้บิตพิเศษมาช่วยเฝ้าระวังและซ่อมแซมข้อมูลที่อาจเพี้ยนไปจากปัจจัยทางไฟฟ้าหรือสิ่งแวดล้อมครับ
